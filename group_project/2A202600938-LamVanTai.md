### Task 4 — Chunking & Indexing ✅

**Chunking strategy:** `RecursiveCharacterTextSplitter`
- Lý do: Tài liệu pháp luật có cấu trúc đoạn văn rõ ràng, splitter ưu tiên tách theo `\n\n` rồi `\n` rồi câu
- `chunk_size = 500`, `chunk_overlap = 50`
- Tạo được **260 chunks** từ 7 documents

**Embedding model:** `OpenAI text-embedding-3-small` (1536 dim)
- Lý do: `sentence-transformers + torch` bị treo trên Python 3.14, OpenAI API ổn định và hỗ trợ tiếng Việt tốt

**Vector store:** `ChromaDB` (local, persistent)
- Lý do: Không cần Docker/server, chạy local đơn giản
- Lưu tại: `data/chroma_db/`

### Yêu cầu 2: RAG Evaluation Pipeline (RAGAS) ✅

**Framework:** RAGAS-style (LLM-as-judge · GPT-4o-mini)  
**File:** `group_project/evaluation/eval_pipeline.py`  
**Dataset:** `group_project/evaluation/golden_dataset.json` — **17 cặp Q&A**

**Phân loại câu hỏi:**
- `legal`: 11 câu — về hình phạt, điều luật, cai nghiện
- `news`: 5 câu — về nghệ sĩ bị bắt
- `mixed`: 3 câu — kết hợp cả hai

#### 4 Metrics đánh giá:

| Metric | Định nghĩa |
|--------|-----------|
| **Faithfulness** | Câu trả lời có bám sát context, không hallucinate? |
| **Answer Relevancy** | Câu trả lời có đúng và đầy đủ cho câu hỏi? |
| **Context Recall** | Retriever có lấy đủ evidence từ ground truth? |
| **Context Precision** | Trong context lấy về, bao nhiêu % thực sự hữu ích? |

#### So sánh A/B:

| Config | Mô tả |
|--------|-------|
| **Config A** | Hybrid Search (Semantic + BM25) + Jina Reranking |
| **Config B** | Dense-only (chỉ Semantic Search, không BM25, không rerank) |

#### Phân tích kết quả:

**Điểm mạnh:**
- Answer Relevancy = **0.98** — GPT-4o-mini hiểu câu hỏi rất tốt
- Faithfulness = **0.85** — LLM bám sát nguồn, ít hallucinate
- Câu hỏi về tin tức nghệ sĩ (Q09–Q13) đạt điểm cao hơn câu hỏi pháp luật

**Điểm yếu:**
- Context Recall = **0.75** — Corpus thiếu Bộ luật Hình sự đầy đủ (Điều 248–256)
- Config B Dense-only có Context Recall nhỉnh hơn nhẹ → câu hỏi đơn giản không cần rerank
- Q01, Q02 (hình phạt cụ thể) có điểm thấp nhất do thiếu văn bản pháp luật chi tiết

**Đề xuất cải tiến:**
1. Thêm Bộ luật Hình sự đầy đủ, Nghị định 105/2021 → tăng Context Recall lên 0.90+
2. Chunk theo điều khoản luật thay vì fixed-size (MarkdownHeaderTextSplitter)
3. Dùng `underthesea` tokenizer cho BM25 → tăng Context Precision
4. Fine-tune Jina Reranker trên văn bản pháp luật Việt Nam
5. Knowledge Graph (giai đoạn 2) — liên kết điều luật ↔ nghệ sĩ ↔ vụ án

## V. STACK KỸ THUẬT

| Component | Công nghệ | Lý do |
|-----------|-----------|-------|
| Embedding | OpenAI text-embedding-3-small | sentence-transformers treo trên Python 3.14 |
| Vector Store | ChromaDB | Local, không cần Docker |
| Chunking | LangChain RecursiveCharacterTextSplitter | An toàn với mọi loại văn bản |
| Lexical Search | rank-bm25 (BM25Okapi) | Bắt keyword chính xác tốt |
| Merge | RRF (Reciprocal Rank Fusion) | Không cần normalize score giữa 2 ranker |
| Reranking | Jina jina-reranker-v2-base-multilingual | Multilingual, tốt cho tiếng Việt |
| Fallback | PageIndex Vectorless RAG | Không cần vector, dùng LLM tree search |
| LLM | OpenAI gpt-4o-mini | Nhanh, rẻ, đủ tốt cho RAG |
| UI | Streamlit | Nhanh, dễ dùng cho demo |
| Evaluation | RAGAS-style (LLM-as-judge) | RAGAS package bị conflict Python 3.14 |
| Web Crawling | Playwright + BeautifulSoup | Crawl4AI không tương thích Python 3.14 |
